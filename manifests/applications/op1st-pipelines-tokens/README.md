# op1st-pipelines-tokens

Single source of truth for tokens / credentials consumed by op1st Pipelines
(Tekton + Pipelines-as-Code). Canonical SealedSecrets live in `op1st-gitops`
and are fanned out to consumer namespaces by emberstack/reflector.

## Contents

| Secret name | Type | Purpose | Mirrored to |
|---|---|---|---|
| `codeberg-pac-token` | `Opaque` | PaC API token + webhook secret for the `op1st-gitops` Codeberg user. Keys: `provider.token`, `webhook.secret`. | `op1st-pipelines`, `b4mad-machdenstaat-stage`, `feeldata-dev` |
| `codeberg-pusher` | `kubernetes.io/dockerconfigjson` | Codeberg container-registry push credentials. | `op1st-pipelines` |

Add new tokens here as they appear.

## Architecture

```
op1st-gitops/<secret>          <-- canonical SealedSecret (this app)
        |
        |  emberstack/reflector auto-mirrors per
        |  reflection-auto-namespaces annotation
        v
<target-ns>/<secret>
```

Consumers reference the mirrored Secret by name (e.g., PaC `Repository` CRs
set `git_provider.secret.name: codeberg-pac-token`).

## Files

- `<secret>.enc.yaml` — SOPS-encrypted plaintext `Secret`. Source for
  `kubeseal`. Never applied to the cluster directly.
- `<secret>.yaml` — `SealedSecret` consumed by ArgoCD. Regenerated by
  `seal.sh` from the file above.
- `seal.sh` — rotation helper. Iterates every `*.enc.yaml`, runs `kubeseal`,
  re-merges the plaintext `metadata.annotations` onto
  `spec.template.metadata.annotations` (kubeseal strips them otherwise).

## Rotation procedure

Prereqs: `sops`, `kubeseal`, `yq` (mikefarah v4+), cluster admin kubeconfig,
valid GPG key listed in `.sops.yaml`.

```bash
cd manifests/applications/op1st-pipelines-tokens

# 1. Edit plaintext (decrypts, opens $EDITOR, re-encrypts + recomputes MAC).
sops codeberg-pac-token.enc.yaml

# 2. Reseal every *.enc.yaml in this directory.
./seal.sh

# 3. Commit + push.
git add ./*.yaml ./*.enc.yaml
git commit -m "chore(secrets): rotate <secret-name>"
git push
```

`seal.sh` environment overrides (rarely needed):

- `SEALED_SECRETS_CONTROLLER_NS` (default `sealed-secrets`)
- `SEALED_SECRETS_CONTROLLER_NAME` (default `sealed-secrets-controller`)

ArgoCD picks up the new `SealedSecret`, sealed-secrets controller writes the
unsealed `Secret`, reflector fans out to consumer namespaces. PaC controller
picks up the new token on the next webhook event — no restart needed.

## Minting a fresh Codeberg API token

1. Log in as `op1st-gitops` on https://codeberg.org/.
2. Settings -> Applications -> Manage Access Tokens -> Generate New Token.
3. Name: `op1st-pac-<YYYY-MM-DD>`.
4. Scopes: `read:repository`, `write:repository`, `write:issue`.
5. Paste the token value into `codeberg-pac-token.enc.yaml` field
   `stringData.provider.token` and run the rotation steps above.

## codeberg-pusher initial migration (one-off)

The `codeberg-pusher` SealedSecret was relocated from
`manifests/applications/op1st-pipelines/secrets/` into this directory. Its
ciphertext is bound to the old `(op1st-pipelines, codeberg-pusher)` tuple by
sealed-secrets' default strict scope and is rejected at
`(op1st-gitops, codeberg-pusher)`. The file is staged with
`encryptedData: REPLACE_ME_VIA_KUBESEAL` until the first reseal.

To complete the migration:

```bash
cd manifests/applications/op1st-pipelines-tokens

# Open plaintext, change metadata.namespace from op1st-pipelines to
# op1st-gitops, add reflector annotations matching codeberg-pac-token.
sops codeberg-pusher.enc.yaml

./seal.sh
git add codeberg-pusher.{enc.,}yaml
git commit -m "chore(secrets): reseal codeberg-pusher for op1st-gitops"
```

## Consumer migration

After this app is healthy, update consumers to reference the mirrored Secret
in their own namespace by name (no rename — names stay
`codeberg-pac-token` / `codeberg-pusher`). For PaC `Repository` CRs:

```yaml
spec:
  git_provider:
    secret:
      name: codeberg-pac-token   # was: codeberg-org-op1st-pipelines
      key: provider.token
    webhook_secret:
      name: codeberg-pac-token
      key: webhook.secret
```

Track per-consumer migration in a separate PR. Old hand-patched Secrets stay
in place until all consumers are cut over, then get removed.
